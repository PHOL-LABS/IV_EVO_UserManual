\chapter{Typical setup cases}\label{ch:setup-cases}

Always establish a stable MODBUS session (no red error banners) before applying any of the following adjustments. The screenshots originate from the IV-Conf web pack but the Android beta offers equivalent controls.

\section{Colour adjustments}
\begin{enumerate}
    \item Click \texttt{Get data} to read the current preset.
    \item Change the hue controls to match the desired display colour. The preview text mirrors the indicator output.
    \item Adjust the \texttt{Bl} backlight slider to tune the background intensity.
    \item Press \texttt{Send data} and wait for the transfer to finish.
\end{enumerate}

\section{Range calibration}
\begin{enumerate}
    \item Retrieve the configuration with \texttt{Get data}.
    \item Update \texttt{Range min} and \texttt{Range max} so the indicator reflects the true sensor span (voltmeter, barometer, lambda, or boost).
    \item Send the data and verify the live reading against a trusted meter.
\end{enumerate}

\section{Thermometer tuning}
\begin{enumerate}
    \item Measure the sensor resistance with a multimeter and compare it to the values in \autoref{app:temperature-table}. Replace the probe if it falls outside tolerance.
    \item Use \texttt{Get data} to load the current parameters.
    \item Enter the appropriate beta coefficient and nominal resistance for the custom NTC sensor.
    \item Upload the changes with \texttt{Send data}.
\end{enumerate}

\section{Switching bar and dot modes}
\begin{enumerate}
    \item Fetch the configuration via \texttt{Get data}.
    \item Set \texttt{Bar mode} to 1 for a bar-style display or clear it for dot indication.
    \item Press \texttt{Send data} to commit the change.
\end{enumerate}
